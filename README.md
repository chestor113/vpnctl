Данный проект является пэт-проектом, для освежения в памяти и закрепления питона.
В наше тяжелое время, когда ВПН стал неотъемлимой частью жизни, стабильно работающий ресурс стал очень востребованным и когда семья, друзья, знакомые узнают, что я не испытываю проблем с этим, просят поделиться.

Да, есть некоторые готовые вебморды для ВПНов. Но я решил воспользоваться моментом и поучиться писать код.

ИИ я использовал в качестве ментора для подсказок и доработок, применения более красивых подходов, до которых в силу отсутствия опыта я не смог бы сам додуматься.

Основная задумка, это через консоль создавать юзеров с автоматическим генерированием конфитгураций юзерской части и клиентской. 
В настоящий момент генерируется только WireGuard, но будет параллельно генерироваться сразу xRay.  
В качестве источника правды используется SQLLite.   
В БД хранятся различные атрибуты(uuid, username, telegram, is_enabled, expires_at и другие)  
На настоящий момент реализованы команды:  
- create -u --username username -t --telegram telegram -d --days --tag  
- renew -t --telegram -d --days   
- disable -t --telegram  
- enable -t --telegram 
- list --all (Выводит всех пользователей) -a (Выводит только активных) -i (Выводит только выключенных)
- delete -t --telegram

Файлы конфигурации юзеров и сервера хранятся в папке Wireguard/ 

В планах автоматизировать выключение истекших ВПН пользователей по расписанию.  
Дописать автоконфигурирование конфиги для xray.  

Структура проекта  
services.py — бизнес-логика  
db.py — работа с SQLite  
wg_config.py — генерация клиентских конфигов  
wg_server_config.py — сборка server config  
cli.py — CLI интерфейс  
config.py - функции чтения секретов  
result.py - датакласс для унифицированного возврата значений функций  
access_grants.sql - скрипт для создания используемой БД  
logger_config.py - конфигурирование логгирования.
